# Historias de Usuario para el Sistema de Gestión CAT

Estas historias describen las funcionalidades clave del sistema desde la perspectiva de los usuarios.

1. *Como estudiante*, quiero registrar una solicitud de soporte técnico, para reportar problemas con mi equipo rápidamente.
2. *Como técnico*, quiero asignar prioridades a las solicitudes, para atender primero las urgentes.
3. *Como administrador*, quiero generar reportes de solicitudes, para analizar el rendimiento del CAT.
4. *Como docente*, quiero consultar el estado de mis solicitudes, para saber cuándo se resolverán.
5. *Como usuario*, quiero recibir notificaciones por email, para estar al tanto de actualizaciones en mis tickets.
6. *Como técnico*, quiero actualizar el estado de una solicitud, para mantener trazabilidad.
7. *Como administrador*, quiero asignar solicitudes a técnicos específicos, para distribuir la carga de trabajo.
8. *Como usuario*, quiero adjuntar archivos a mi solicitud, para proporcionar evidencia del problema.
9. *Como administrador*, quiero filtrar solicitudes por fecha o tipo, para gestionar mejor el backlog.
10. *Como técnico*, quiero cerrar una solicitud con comentarios, para registrar la solución aplicada.
